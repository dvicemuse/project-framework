Controllers

Controller Basics

A controller is a class file that is named in a way that it can be associated with a URI.

For example, if you request:

mysite.com/blog/index/

The framework would attempt to load the controller class Blog_Controller, and call the function Blog_Controller:index().

The "index" function is loaded by default if the second segment of the URI is empty. The request above could be shortened to:

mysite.com/blog/

Naming Guidelines

  • One word
  • Begin with a capital letter

Anatomy of a Controller

Controller classes are stored in the framework/controller/ folder.

Loading a Helper

Models can be called from anywhere within the framework construct:

$this->load_helper('Helper');

Once loaded, you can access your helper functions using an object with the same name as your class:

$this->load_helper('Paginate');
$this->Paginate->function();

The load_helper() function returns the helper class object, enabling method chaining. The above example could be rewritten as:

$this->load_model('Paginate')->function();
